Security Policy

Last updated: June 15, 2025

Introduction

The Airport Authority of India (AAI) Asset Management System is designed with security as a fundamental principle. This Security Policy outlines the measures we implement to protect our systems, data, and users from security threats.

Security Alert: Any suspected security incidents should be reported immediately to the AAI Security Operations Center at soc@aai.gov.in or by calling +91-11-2467-SECU (7328).

System Security Measures

We implement multiple layers of security to protect the AAI Asset Management System:

Access Control

  • RBAC Role-Based Access Control for all system functions
  • MFA Multi-Factor Authentication for all user accounts
  • JIT Just-In-Time access provisioning for privileged accounts
  • Regular access rights reviews and recertification

Data Protection

  • TLS 1.3 Encryption of all data in transit
  • AES-256 Encryption for sensitive data at rest
  • Data masking for sensitive information in logs and reports
  • Regular data backup with air-gapped storage

Network Security

  • Firewalls with application-layer filtering
  • Intrusion Detection and Prevention Systems (IDPS)
  • Network segmentation and micro-segmentation
  • Continuous network traffic monitoring

User Responsibilities

All users of the AAI Asset Management System must adhere to the following security practices:

Note: Failure to comply with these security requirements may result in disciplinary action under AAI policies.

Account Security

  • Never share your login credentials with anyone
  • Use strong, unique passwords (minimum 12 characters)
  • Immediately report any suspicious account activity
  • Lock your workstation when leaving your desk

Data Handling

  • Only access data required for your job function
  • Never store sensitive data on personal devices
  • Follow data classification and handling procedures
  • Report any suspected data breaches immediately

Physical Security

  • Never allow tailgating at secure access points
  • Challenge unbadged individuals in secure areas
  • Secure all devices when not in use
  • Follow clean desk policies

Incident Response

AAI maintains a comprehensive incident response plan for security events:

  • Detection: 24/7 monitoring by Security Operations Center
  • Containment: Immediate isolation of affected systems
  • Investigation: Forensic analysis and root cause determination
  • Eradication: Removal of threats and vulnerabilities
  • Recovery: Secure restoration of systems and data
  • Lessons Learned: Process improvement and training

Vulnerability Management

We maintain a proactive vulnerability management program:

  • Monthly vulnerability scanning of all systems
  • Quarterly penetration testing by certified ethical hackers
  • Patch management within 7 days for critical vulnerabilities
  • Continuous monitoring of threat intelligence feeds

Compliance Standards

The AAI Asset Management System complies with the following standards:

  • ISO/IEC 27001:2022 Information Security Management
  • National Critical Information Infrastructure Protection Centre (NCIIPC) guidelines
  • Cert-In (Indian Computer Emergency Response Team) directives
  • AAI Information Security Policy

Security Training

All AAI employees receive regular security awareness training:

  • Mandatory annual security awareness training
  • Quarterly phishing simulation exercises
  • Role-specific security training for system administrators
  • Secure coding training for developers
